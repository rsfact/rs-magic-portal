# サービス層ルール

- ビジネスロジックをカプセル化する。
- 型定義（Pydantic）により入出力を保証する。descriptionは不要、exampleは必須。
- インプットは基本的に個別引数。
    - JSONを受け取る場合、モデル名はInFunctionNameなどとする。
- アウトプットは必ず定義済みモデルで返却。
    - モデル名は、OutFunctionNameなどとする。
    - @validate_callで、関数呼び出しの入出力を保証する。
- 外部サービスのみならず、標準入出力するコマンドラインアプリケーションも同様にserviceに固める。
- テスト: 同ディレクトリにテストファイルを生成（例: test_ffmpeg_service.py）
- 基本的に、エラーハンドリングは行わない。
    - ユースケース層で吸収する。
    - 個別にハンドリングしたい場合には特別に実装する。
- ユースケースレベルの関数名は create, get, search, delete など簡潔にする。
- 認可が必要な処理には引数に user (Userモデルオブジェクト) を渡し、関数内で所有権などのチェックを必ず行う（他のユーザーのデータを操作させない）。
