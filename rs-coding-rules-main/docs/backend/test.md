# テストルール

## ユニットテスト

- serviceにのみテストを記述する。
- モックを避け、実操作する。
- クラス名は `Test` とする。
- 正常系を優先し、エッジケースは最低限に留める
- setup_method, teardown_method により状態管理・後片付けする。
- テストファイルは、serviceファイルと同じディレクトリに配置する。
- 命名は、 `test_ffmpeg_serivce.py` などとする。 
- 1ファイルあたり300行未満とすること。

## 統合テスト

- tests ディレクトリ以下に、シナリオごとに作成する。
- クラス名はTestとして、1ファイルあたり複数シナリオを抱えられる。
- 最もシンプルに記述し、エッジケースの検証は最低限とする。
